# 高位交叉编址
如上图，左边的编址方式就是高位交叉编址，即将最高的两位当做存储体的编号。
由于存储器需要刷新时间并且这种编址方式，相邻的地址位于相同的存储体中，所以，当我们连续访问地址的时候，必须要等待刷新完毕才能再次访问，这就导致了时间的浪费的。
# 低位交叉编址
如上图右边的方式，将一个存储单元的低位作为存储体的编号。
